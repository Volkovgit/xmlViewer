Как организовать кнопки с контекстными действиями?

Выберите подход для отображения кнопок в зависимости от типа файла

A

Динамические кнопки

Кнопки появляются/исчезают в зависимости от активного файла

Слева: боковая панель действий
Когда выбран .xsd файл:
Actions for XSD
📊 Show Graph
📄 Generate XML
✓ Validate
Когда выбран .xml файл:
Actions for XML
📋 Generate XSD
🔗 Assign Schema
✓ Validate XSD

✓ Плюсы

  • Только нужные кнопки
  • Чистый интерфейс
  • Меньше путаницы

✗ Минусы

  • Кнопки "прыгают"
  • Нужно запоминать где что
B

Группировка по типу

Все кнопки видны, но сгруппированы с визуальным разделением

Слева: боковая панель действий
XSD Actions
📊 Show Graph
📄 Generate XML
✓ Validate
XML Actions
📋 Generate XSD
🔗 Assign Schema
✓ Validate XSD
* Неприменимые кнопки затемнены

✓ Плюсы

  • Все действия на виду
  • Явная группировка
  • Легко найти нужное

✗ Минусы

  • Больше места занимает
  • Визуальный шум
C

Гибрид: быстрые действия + меню

Основные действия всегда видны, остальные в выпадающем меню

Слева: боковая панель действий
📊 Show Graph
Generate & Validate ▼
📄 Generate XML
📋 Generate XSD
✓ Validate Schema
File Actions ▼
💾 Save
💾 Save As
📂 Open File

✓ Плюсы

  • Компактно
  • Главное на виду
  • Легко расширять

✗ Минусы

  • Нужен клик для действий
  • Меньшая наглядность
